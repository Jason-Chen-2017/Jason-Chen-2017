🏗️ 第 1 章

OpenClaw 架构深度解析

核心组件 × 执行流程 × 记忆系统 × 安全机制 = 构建稳定可靠的 AI 自动化系统

1.1 OpenClaw 核心架构概览

💡 本章学习目标：
1. 理解 OpenClaw 的四大核心组件及其作用
2. 掌握任务执行的完整流程和关键节点
3. 了解记忆系统的工作原理和配置方法
4. 熟悉安全机制和权限控制策略

OpenClaw 之所以能够成为"真正能干活"的 AI 智能体，关键在于其精心设计的架构。让我们深入内部，看看它是如何工作的。

🔧 OpenClaw 核心架构图
1
感知层（Perception Layer）
负责接收用户指令，支持多种输入方式：命令行、IM 消息（微信/飞书/钉钉）、API 调用、定时触发器等。这一层就像人的眼睛和耳朵，时刻监听各种输入信号。
2
决策层（Decision Layer）
核心是大语言模型（LLM），负责理解指令意图、拆解复杂任务、规划执行步骤。这是 OpenClaw 的"大脑"，决定要做什么、怎么做。
3
执行层（Execution Layer）
包含各种 Skills（技能模块），如文件操作、浏览器控制、代码执行、API 调用等。这是 OpenClaw 的"手脚"，真正动手完成任务。
4
记忆层（Memory Layer）
存储用户偏好、历史任务、上下文信息，让 AI 越用越懂你。包括短期记忆（当前会话）和长期记忆（持久化存储）。

1.2 任务执行流程详解

当你对 OpenClaw 说"帮我整理本周工作日报"时，背后发生了什么？让我们一步步拆解：

📊 完整执行流程：
  1. 指令接收：通过飞书收到消息"帮我整理本周工作日报"
  2. 意图识别：LLM 分析这是一个"报告生成"任务，需要收集工作数据并格式化输出
  3. 任务拆解：
    • 步骤 1：从日历获取本周会议记录
    • 步骤 2：从 Git 获取代码提交记录
    • 步骤 3：从文档文件夹获取本周创建的文档
    • 步骤 4：整合信息，生成结构化日报
    • 步骤 5：通过飞书发送给用户
  4. 顺序执行：按顺序调用相应 Skills 完成每个步骤
  5. 结果反馈：将生成的日报发送给用户，并询问是否需要修改
  6. 记忆存储：将本次任务的执行过程和用户反馈存入长期记忆

关键执行机制

机制 作用 配置位置
任务队列 管理并发任务，避免资源冲突 config/task_queue.yml
错误重试 失败任务自动重试，提高成功率 config/retry_policy.yml
超时控制 防止任务无限期挂起 config/timeout.yml
执行日志 记录详细执行过程，便于调试 logs/execution.log

1.3 记忆系统：让 AI 越用越聪明

OpenClaw 的记忆系统是其区别于普通聊天机器人的关键特性。它不仅能记住你说过的话，还能记住你的偏好、习惯和工作方式。

🧠 记忆系统三层架构
1
短期记忆（Session Memory）
存储当前会话的上下文，对话结束后清除。用于保持对话连贯性，例如你问"它是什么"，AI 知道"它"指代上文提到的对象。
位置：内存中，会话结束后释放
2
长期记忆（Long-term Memory）
持久化存储在本地文件中，包括：
  • 用户偏好（如常用工具、工作时间）
  • 历史任务及执行结果
  • 重要信息（如项目名称、团队成员）

位置：~/.openclaw/memory/long_term.json
3
技能记忆（Skill Memory）
记录已安装的 Skills、使用频率、执行成功率等元数据，用于优化技能推荐和执行策略。
位置：~/.openclaw/skills/metadata.json

1.4 安全机制与权限控制

⚠️ 安全警告：
OpenClaw 拥有强大的系统操作能力，必须配置严格的安全策略，防止误操作或恶意利用。

四层安全防护

层级 防护措施 配置示例
权限隔离 最小权限原则，仅授予必要权限 禁止访问敏感目录（如/etc、系统文件）
操作审计 记录所有操作日志，可追溯 logs/audit.log 记录每条命令执行
危险操作确认 删除、格式化等操作需二次确认 config/safety.yml 设置确认规则
沙箱隔离 在受限环境中执行不确定代码 Docker 容器或系统沙箱

安全配置文件示例

# ~/.openclaw/config/safety.yml
safety:
  # 禁止访问的目录
  forbidden_paths:
    - "/etc"
    - "/root"
    - "/sys"
    - "/proc"
  
  # 需要确认的危险操作
  require_confirmation:
    - "rm -rf *"
    - "mkfs.*"
    - "dd if=.*"
    - "chmod 777.*"
  
  # 允许的命令白名单
  allowed_commands:
    - "ls"
    - "cd"
    - "cat"
    - "grep"
    - "find"
    - "python3"
    - "npm"
  
  # 网络访问限制
  network_restrictions:
    allowed_domains:
      - "api.github.com"
      - "openai.com"
      - "anthropic.com"
    blocked_ports:
      - 22   # SSH
      - 23   # Telnet
      - 3389 # RDP
  
  # 审计日志
  audit:
    enabled: true
    log_file: "~/.openclaw/logs/audit.log"
    log_level: "INFO"  # DEBUG, INFO, WARNING, ERROR

A. 常用工作流模板速查表

# 模板 1：日报自动生成工作流
name: daily_report
trigger: "cron: 0 18 * * 1-5"  # 工作日每天 18 点
steps:
  - action: fetch_calendar
    params:
      date_range: "today"
      include_meetings: true
  - action: fetch_git_commits
    params:
      repo: "~/projects/*"
      date_range: "today"
  - action: generate_report
    template: "daily_report.md"
  - action: send_message
    channel: "feishu"
    recipient: "user"
    
# 模板 2：邮件自动分类工作流
name: email_classifier
trigger: "new_email"
steps:
  - action: analyze_email
    model: "deepseek-chat"
  - action: classify
    categories: ["工作", "订阅", "推广", "个人"]
  - action: move_to_folder
    mapping:
      "工作": "Inbox/Work"
      "订阅": "Inbox/Subscriptions"
      "推广": "Inbox/Promotions"
      "个人": "Inbox/Personal"
  - action: notify_if_urgent
    condition: "priority == 'high'"
    
# 模板 3：竞品监控工作流
name: competitor_monitor
trigger: "cron: 0 9 * * *"  # 每天早上 9 点
steps:
  - action: scrape_website
    urls:
      - "https://competitor1.com/products"
      - "https://competitor2.com/pricing"
  - action: extract_data
    fields: ["product_name", "price", "features"]
  - action: compare_with_db
    database: "competitors.db"
  - action: alert_if_changed
    threshold: "price_change > 10%"
  - action: update_dashboard
    dashboard: "competitor_analysis"

B. 一键部署脚本汇总

#!/bin/bash
# 腾讯云一键部署脚本（Ubuntu 24.04）

echo "🚀 开始部署 OpenClaw..."

# 1. 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 2. 安装 OpenClaw
npm install -g openclaw@latest

# 3. 创建配置目录
mkdir -p ~/.openclaw/{config,memory,skills,logs}

# 4. 下载默认配置
wget https://raw.githubusercontent.com/OpenClaw/templates/main/config.yml \
  -O ~/.openclaw/config/config.yml

# 5. 启动服务
openclaw onboard --install-daemon

# 6. 配置防火墙
sudo ufw allow 18789/tcp
sudo ufw enable

echo "✅ 部署完成！访问 http://$(hostname -I | awk '{print $1}'):18789"
echo "📚 查看日志：tail -f ~/.openclaw/logs/execution.log"
