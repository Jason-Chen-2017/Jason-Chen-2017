Chapter 01

项目简介

了解 OpenClaw 是什么，它能为你做什么

什么是 OpenClaw

OpenClaw 是一款开源自托管的个人 AI 智能体网关（Personal AI Agent Gateway）， 本质是运行在用户自有设备上的本地优先的 AI 代理系统。它可以通过日常通讯工具 （如 Telegram、WhatsApp、Discord 等）接收指令，自动完成文件整理、代码部署、 日程同步等系统级任务，所有数据本地存储，确保隐私零泄露。

💡 项目名称由来
OpenClaw 前身是 2025 年底由奥地利开发者 Peter Steinberger 创立的 "Clawdbot" 项目。 由于名称与 Anthropic 的 "Claude" 相似，2026 年 1 月先后更名为 Moltbot， 最终定名为 OpenClaw。"Open" 凸显开源开放特质，"Claw" 保留龙虾文化符号。

截至 2026 年 2 月，OpenClaw 在 GitHub 上已获得超过 15 万 Star， 成为最受关注的开源 AI 项目之一。

核心特性

🔒 本地优先
所有数据存储在本地设备，不上传云端，隐私完全可控
🌐 全渠道交互
支持 Telegram、WhatsApp、Discord、Slack 等主流通讯工具
⚡ 系统级执行
可直接操作文件系统、执行命令、调用 API，能力无边界
🔌 插件扩展
丰富的插件生态，支持自定义技能扩展

适用场景

  • 个人效率助手：自动整理文件、管理日程、提醒待办
  • 开发者工具：代码部署、日志监控、自动化测试
  • 智能家居中枢：联动 IoT 设备、场景自动化
  • 远程运维：服务器监控、故障告警、批量操作
  • 数据自动化：定时抓取、数据清洗、报告生成
Chapter 02

快速安装

三种主流安装方式，选择最适合你的

前置要求

硬件配置

配置项 最低要求 推荐配置 说明
CPU 4 核 64 位 6 核及以上 Intel i5 / AMD Ryzen 5+
内存 8GB 16GB 本地运行大模型需 16GB+
存储 10GB 20GB+ 存储依赖、模型及日志
网络 稳定连接 100Mbps+ 国内需配置代理

软件环境

组件 版本要求 获取方式
Node.js ≥22.0.0 nodejs.org
Docker ≥20.10.0 docker.com
Git 任意稳定版 git-scm.com
⚠️ 系统要求
Windows：需启用 WSL2 + Linux 子系统（推荐 Ubuntu 22.04）
Mac：macOS 12+（Intel/Apple Silicon 原生支持）
Linux：Debian/Ubuntu 20.04+（兼容性最佳）

方式一：Docker 部署（推荐）

跨平台通用，最适合新手，环境隔离，一键启动。

  1. 安装 Docker 并启动

    验证安装：docker --version

  2. 克隆源码
    bash 
    # 克隆仓库
git clone https://github.com/clawdbot/clawdbot.git
cd clawdbot

# 复制配置文件
cp .env.example .env

# 编辑配置（修改 API Key 等）
vim .env
  3. 配置环境变量

    编辑 .env 文件，修改以下关键配置：

    .env 
    # 自定义 AI 名称
CLAWS_BOT_NAME="MyOpenClaw"

# 服务端口（默认 8080）
CLAWS_SERVER_PORT=8080

# AI 模型（本地填 llama3:8b，云端填 API Key）
CLAWS_DEFAULT_MODEL="llama3:8b"

# 数据存储路径
CLAWS_DATA_DIR="/path/to/your/data"

# Telegram Bot Token（可选）
TELEGRAM_BOT_TOKEN="123456:ABC-DEF1234ghIkl"
  4. 启动容器
    bash 
    # 构建并启动（首次需 5-10 分钟）
docker-compose up -d --build

# 查看状态
docker ps

# 查看日志
docker logs clawdbot
  5. 验证部署

    浏览器访问 http://localhost:8080，出现管理界面即成功。

方式二：一键脚本（Mac/Linux）

原生系统快速部署，无需 Docker。

bash 
# 执行一键安装脚本
curl -fsSL https://clawd.bot/install.sh | bash

# 运行配置向导
clawdbot onboard --install-daemon

# 按向导提示完成：
# - 选择启动模式（推荐 QuickStart）
# - 输入 AI 模型 API Key
# - 选择聊天渠道并输入 Token
🪟 Windows 用户
使用 PowerShell 运行：
iwr -useb https://clawd.bot/install.ps1 | iex

方式三：NPM 安装（进阶）

适合需要灵活配置的开发者。

bash 
# 安装 Node.js 22+（已安装可跳过）
nvm install 22
nvm use 22

# 全局安装 OpenClaw
npm install -g clawdbot@latest

# 启动配置向导
clawdbot onboard

# 启动网关服务
clawdbot gateway --port 18789 --verbose
Chapter 03

配置指南

完成关键配置，让 OpenClaw 真正为你工作

环境变量配置

OpenClaw 的核心配置通过环境变量或 JSON 配置文件管理。

.env / clawdbot.json 
# ===== 基础配置 =====
CLAWS_BOT_NAME="MyAssistant"
CLAWS_SERVER_PORT=8080
CLAWS_DATA_DIR="/home/user/clawdbot/data"

# ===== AI 模型配置 =====
CLAWS_DEFAULT_MODEL="llama3:8b"
OPENAI_API_KEY="sk-..."
ANTHROPIC_API_KEY="sk-ant-..."

# ===== 聊天工具配置 =====
TELEGRAM_BOT_TOKEN="123456:ABC-DEF..."
DISCORD_BOT_TOKEN="..."

# ===== 代理配置（国内用户）
PROXY_URL="http://127.0.0.1:7890"

AI 模型配置

OpenClaw 支持多种 AI 模型，包括本地模型和云端 API。

云端模型（推荐新手）

模型 配置方式 适用场景
Claude 3.5 ANTHROPIC_API_KEY 复杂任务、代码生成
GPT-4 OPENAI_API_KEY 通用任务
智谱 GLM-4 自定义配置 国内用户、中文优化
MiniMax 自定义配置 长文本处理

本地模型（隐私优先）

bash 
# 使用 Ollama 运行本地模型
ollama pull llama3:8b

# 配置 OpenClaw 使用本地模型
CLAWS_DEFAULT_MODEL="llama3:8b"
OLLAMA_HOST="http://localhost:11434"
⚠️ 本地模型要求
运行 8B 参数模型至少需要 16GB 内存，建议使用量化版本（如 llama3:8b-q4_0）。 重度使用场景推荐云端模型。

聊天工具对接

以 Telegram 为例（最稳定、最简单）。

  1. 创建 Telegram Bot

    在 Telegram 中搜索 @BotFather，发送 /newbot 按提示创建。

  2. 获取 Bot Token

    BotFather 会返回 API 密钥，格式如：123456:ABC-DEF1234ghIkl-...

  3. 配置 Token
    .env 
    TELEGRAM_BOT_TOKEN="123456:ABC-DEF1234ghIkl-..."
  4. 重启服务
    bash 
    # Docker 部署
docker-compose restart

# 脚本安装
clawdbot gateway restart
  5. 验证对接

    在 Telegram 中搜索你的机器人，发送 /start，收到欢迎语即成功。

✅ 支持的平台
Telegram WhatsApp Discord Slack 微信（企业微信）

基础指令示例

通过自然语言下达指令，AI 会自动理解并执行。

功能类型 指令示例 说明
文件管理 "整理下载文件夹，按类型分类" 自动创建子文件夹并分类
代码执行 "运行 test.py，返回结果" 执行脚本并输出

文本处理 "将会议录音转成文字纪要" 需对接语音 API
系统监控 "监控 CPU，超 80% 提醒我" 主动告警功能

文件管理实战

自然语言指令 
# 指令 1：按类型分类
"把下载文件夹里的文件按类型整理，PDF 放一起，图片放一起"

# 指令 2：清理旧文件
"删除下载文件夹里超过 30 天的文件"

# 指令 3：批量重命名
"把照片文件夹里的文件按日期重命名"

# 指令 4：查找文件
"帮我找一下上周修改的 Excel 文件"
💡 最佳实践
首次使用建议先在测试目录尝试，确认无误后再对重要目录操作。 可以设置白名单目录，防止误删重要文件。

代码执行实战

自然语言指令 
# 指令 1：运行脚本
"运行当前目录下的 deploy.sh 脚本"

# 指令 2：Git 操作
"提交当前修改，commit 信息是'修复登录 bug'"

# 指令 3：Docker 部署
"构建 Docker 镜像并启动容器"

# 指令 4：查看日志
"显示最近 100 行应用日志，筛选 ERROR 关键字"
🔒 安全警告
OpenClaw 拥有系统级权限，禁止在主力电脑直接运行！ 建议使用闲置设备或云服务器，并配置命令白名单。

自定义 Agent

创建专属的 AI 智能体，定制特定领域能力。

clawdbot.json 
{
  "agents": {
    "defaults": {
      "model": { "primary": "glm/glm-4-plus" },
      "tools": ["file_manager", "shell_executor"]
    },
    "custom_agent": {
      "name": "代码助手",
      "model": { "primary": "claude-3-5-sonnet" },
      "tools": ["git_ops", "code_runner", "test_executor"],
      "restrictions": {
        "allowed_dirs": ["/home/dev/projects"],
        "blocked_commands": ["rm -rf", "sudo"]
      }
    }
  }
}

安全加固

🛡️ 安全配置清单
  • ✅ 配置命令白名单，禁止危险命令
  • ✅ 设置允许操作的目录范围
  • ✅ 启用操作日志审计
  • ✅ 配置二次确认机制（删除文件等）
  • ✅ 定期备份重要数据
  • ✅ 使用专用账户运行，限制权限
security.json 
{
  "security": {
    "command_whitelist": [
      "ls", "cat", "grep", "find",
      "git", "npm", "docker"
    ],
    "command_blacklist": [
      "rm -rf", "sudo", "chmod 777"
    ],
    "allowed_directories": [
      "/home/user/projects",
      "/home/user/downloads"
    ],
    "require_confirmation": [
      "delete", "move", "execute"
    ],
    "audit_log": true,
    "max_execution_time": 300
  }
}
Chapter 06

常见问题

快速排查与解决方案

安装问题

❌ 问题：容器启动失败，提示依赖缺失
原因：Docker 版本过低
解决：升级 Docker 至 20.10+，执行 docker-compose down -v 清除缓存后重试
❌ 问题：Node.js 版本不兼容
原因：版本低于 22.0.0
解决：nvm install 22 && nvm use 22
❌ 问题：源码克隆失败
原因：网络限制
解决：直接下载 GitHub 压缩包，或配置代理

运行问题

⚠️ 问题：Telegram 机器人无响应
原因：API 密钥错误或网络无法访问
解决：核对密钥，国内用户配置代理，在 .env 添加 PROXY_URL="http://127.0.0.1:7890"
⚠️ 问题：无法读写本地文件
原因：权限不足
解决：Docker 部署需映射目录并执行 chmod 777 /path/to/dir
⚠️ 问题：运行卡顿
原因：硬件算力不足
解决：关闭本地大模型，改用云端模型；或降低模型参数

性能优化

  • 降低响应延迟：使用云端模型替代本地模型
  • 减少内存占用：限制并发任务数，配置 MAX_CONCURRENT_TASKS=3
  • 加快启动速度：使用 SSD 存储，预加载常用模型
  • 优化网络请求：配置本地缓存，减少重复 API 调用
  • 延长设备寿命：设置电脑"永不睡眠"，WSL2 用户定期执行 wsl --shutdown
💡 成本优化建议
轻度使用选择国产模型（如 GLM-4.5），重度使用搭配 Claude Opus。 本地可运行小模型处理简单任务，复杂任务路由到云端。
